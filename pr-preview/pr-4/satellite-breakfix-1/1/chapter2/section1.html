Guided Solution

Objectives

  • Investigate why the RHEL 8 server upgrade to RHEL 9 via Leapp is failing in Red Hat Satellite 6.

  • Solve the upgrade issue in the hands-on lab environment.

Instructions

  1. Try to list the available repos on the client and it fails with:

    subscription-manager repos
    Sample output:
    Repositories disabled by configuration.

  2. Change the value of manage_repos in /etc/rhsm/rhsm.conf to 1 instead of 0.

    subscription-manager refresh

  3. Install the leapp utility via dnf install leapp-upgrade and then run leapp preupgrade.

    dnf install leapp-upgrade
leapp preupgrade
    Sample output:
    [ERROR] Actor: target_userspace_creator
Message: Unable to install RHEL 9 userspace packages.

  4. Sync and add the 9.x minor appstream and baseos repos to the content view:

    1. Go to Satellite web ui → Content → Red Hat Repositories

    2. Search with rhel-9-for-x86_64-baseos-rpms and enable baseos for 9.4 minor version or the supported minor version as per this doc.

    3. Click on the + beside 9.4 to enable it.

    4. Go to Content → Sync Status → Expand all

    5. Click on the checkbox for baseos for 9.4 and hit Synchronize Now at the bottom.

  5. Perform the same steps for appstream as well:

    1. Go to Satellite web ui → Content → Red Hat Repositories

    2. Search with rhel-9-for-x86_64-appstream-rpms and enable appstream for 9.4 minor version or the supported minor version as per this doc.

    3. Click on the + beside 9.4 to enable it.

    4. Go to Content → Sync Status → Expand available

    5. Click on the checkbox for appstream for 9.4 and hit Synchronize Now at the bottom.

  6. Now add there 2 repos to the content view:

    1. Content → LifeCycle → Content Views

    2. Click on leapp_8to9 and then on Repositories

    3. Add the 2 repos [baseos and appstream for 9.4] and then Publish a new version of this content view.

  7. leapp preupgrade now fails with

    leapp preupgrade
    Sample output:
    OSError: [Errno 24] Too many open files
sqlite3.OperationalError: unable to open database file

  8. Run the below command to set the ulimit.

    ulimit -n 16384

  9. Once the leapp preupgrade completes with 0 errors and 0 inhibitors , run the leapp upgrade

    leapp upgrade
    Sample output:
    file /usr/lib64/engines-3/afalg.so from install of openssl-libs-1:3.0.7-28.el9_4.x86_64 conflicts with file from package openssl3-libs-3.2.2-2.1.el8.x86_64

  10. The openssl3-libs-3.2.2-2.1.el8.x86_64 rpm is installed from EPEL repo and hence needs to be removed to fix this error.

    yum remove openssl3-libs-3.2.2-2.1.el8.x86_64

  11. Run the leapp upgrade.

    leapp upgrade

  12. Once leapp upgrade is successful and it asks for reboot, reboot the system and confirm that the server is now booted with RHEL9 kernel.
